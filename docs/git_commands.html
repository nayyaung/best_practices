Useful Git Commands

Reapplies commits on top of another base tip.

Used to integrate changes from one branch into another, creating a linear commit history.

git rebase <branch>

git cherry-pick

Description: Apply the changes introduced by some existing commits.

Usage: Used to apply specific commits from one branch to another.

git cherry-pick <commit-hash>

git stash

Description: Stashes the changes in a dirty working directory away.

Usage: Temporarily saves changes that are not ready to be committed.

git stash
git stash apply

git bisect

Description: Uses binary search to find the commit that introduced a bug.

Usage: Helps in identifying the commit that introduced a bug by performing a binary search.

git bisect start
git bisect bad
git bisect good <commit>

git reflog

Description: Shows a log of changes to the local repository's HEAD.

Usage: Useful for recovering lost commits, commits before rebasing and understanding the history of changes.

git reflog

git reset

Description: Resets the current HEAD to a specified state.

Usage: Used to undo changes by resetting the current branch to a previous state.

git reset --soft <commit>
git reset --mixed <commit>
git reset --hard <commit>

git revert

Description: Creates a new commit that undoes the changes made by a previous commit.

Usage: Used to undo changes without rewriting commit history.

git revert <commit>

git filter-branch

Description: Rewrites branches by filtering the commit history.

Usage: Used for rewriting commit history, such as removing sensitive data from all commits.

git filter-branch --tree-filter 'rm -rf secrets/' HEAD

git blame

Description: Shows what revision and author last modified each line of a file.

Usage: Used to identify who made changes to specific lines in a file.

git blame <file>

git show

Description: Shows various types of objects.

Usage: Used to display information about a commit, tag, or other Git object.

git show <commit>

git diff

Description: Shows changes between commits, commit and working tree, etc.

Usage: Used to view differences between various states of the repository.

git diff <commit1> <commit2>

git log

Description: Shows the commit logs.

Usage: Used to view the commit history with various formatting options.

git log --oneline --graph --decorate --all

git tag

Description: Creates, lists, deletes, or verifies a tag object signed with GPG.

Usage: Used to create and manage tags for marking specific points in history.

git tag -a v1.0 -m "Version 1.0"
git push origin v1.0

git clean

Description: Removes untracked files from the working tree.

Usage: Used to clean the working directory by removing untracked files.

git clean -f -d

git remote

Description: Manages set of tracked repositories.

Usage: Used to manage remote repository connections.

git remote -v

git squash

Combines multiple commits into a single commit.

Used to simplify the commit history by merging several commits into one. Squash can be done by merging or by doing reset and commit
